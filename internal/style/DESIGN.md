# Style Sample Design

## 问题起源

Goink 已有「短文提取」功能：用户贴一段样文 → LLM 分析六维度（句式、用词、修辞、节奏、叙事视角、氛围语调）→ 生成仿写 Skill。功能可用，但用户反馈体验太轻量，像一次性小工具。用户期望的是一个能长期积累、看起来更专业的风格管理体系。

原始 issue 原文：「功能越来越完善了……可以参考知识库的模式，做一个增强版，支持批量导入一些小说样本，模仿风格，生成仿写 skill，而不仅仅是某一段样文。」

## 讨论过程

### 探索方向一：全书导入 + 自动分析

最初理解是用户想要导入整本书来分析风格。做了 EPUB/TXT/MD 导入功能，可以自动切章并创建 Novel。然后围绕「整本分析」设计了完整流程：导入 → 建 Novel → 选中 → 遍历全书章节调 LLM → 综合生成 Skill。

**否决原因**：

1. 全书分析对风格提取收益极低。风格是局部特征——句长习惯、用词偏好、对话节奏，三五段代表性段落足以看清。全书第一百章不会提供比前三章更多的风格信号，只会引入情节噪音。

2. Token 消耗与收益严重不匹配。中长篇动辄几十万字，即使抽样也是巨大的上下文负担。这些 token 花在了堆量上，而非提升分析质量。

3. 压缩——即先对全书做摘要再分析——对风格是毁灭性的。摘要把「林雷望着天空喃喃自语：我到底该走哪条路？」变成「林雷感到迷茫」，句式、用词、节奏信息全部丢失。压缩对内容提取是增益，对风格提取是降维打击。

4. 市面上已有项目实现了全书拆解流程，但其核心设计目标是「建立书籍追踪体系」（角色、伏笔、时间线、剧情线）而非「风格蒸馏」。风格产量（文风档案）只是其六阶段管道的副产品之一，且实际只采样了三章做统计。

### 探索方向二：跨书素材库 + 分组管理

考虑过在 Novel 之上建立跨书素材库，支持分组——「金庸武侠」「番茄爽文」「知乎短篇」等标签，同组素材一起蒸馏。

**否决原因**：

1. 设计过度。一个仿写 Skill 对应一种风格，素材必须同源同类。跨书分组是多余的抽象层——用户不会同时分析金庸和海明威来生成一个 Skill。

2. 素材归属模糊。素材存哪个 Novel 下？如果没有分组，随意堆砌会导致后续蒸馏时产生风格冲突——LLM 面对跨风格杂糅的素材会产出四不像的 Skill。

### 选定方案：Novel 级素材池 + 卡片管理

风格素材归属于特定 Novel，存在 `style_samples/` 目录下。每个素材是一个独立文件（YAML frontmatter + 正文），用户可随时添加、编辑、删除。卡片网格展示，勾选后批量蒸馏。

**选择理由**：

1. **积累感**：用户持续喂养素材，池子越来越厚，重新蒸馏时 Skill 质量逐步提升。与知识库的「持续导入 → 积累 → 检索」体验一致。

2. **操作门槛低**：就是贴 → 存 → 勾 → 点。不需要理解分组、标签、全书导入等概念。

3. **与现有架构契合**：素材文件跟 Novel 走 git 版本控制，Skill 也在同一 Novel 下。风格素材和产物归属明确，不引入额外抽象层。

4. **确定性统计加持**：多段素材拼接后，Go 代码可计算句长分布、标点密度、段落节奏等客观指标。这些数据随素材一起喂给 LLM，让分析从「凭感觉」升级为「有数据支撑」。统计是零 token 消耗的确定性计算。

## 核心原则

**片段优于全书**：风格是局部信号，三五段代表性段落的信息密度远高于全书。全书遍历增加的是数量而非质量。

**原文优于摘要**：风格特征活在原文的肌理里——句长、标点、用词顺序。任何形式的压缩都会丢失风格信息。

**统计先行，LLM 后归纳**：句长分布、标点密度、段落节奏这些用 Go 代码算，100% 可靠，零 token。LLM 只做它擅长的事——基于统计数据做风格归纳和仿写要点提炼。

**素材与产物同属一个 Novel**：不建跨书素材库，不引入额外分组维度。简单即正确。

## Storage

```
{DataDir}/novels/{novelID}/style_samples/{id}.md
```

每个文件格式：

```yaml
---
name: 金庸-射雕第一回
tags: [武侠, 金庸, 打斗]
---

风雪中，一名道人...
```

- ID 为纳秒时间戳，全局唯一
- YAML frontmatter 存 name + tags（列表）
- 正文直接写，无需转义
- 文件走 git 版本控制，与章节、skill 同级
- `List()` 扫描目录下所有 `.md` 文件，按创建时间排序
- `Load()` 按 ID 读单文件，解析 frontmatter 后返回完整 Sample

## Stats Engine

`ComputeStats(samples []Sample) Stats` —— 纯 Go 代码，零 LLM 调用，零 token 消耗。

输入是多段素材拼接后的全文，输出 Stats 结构体：

| 指标 | 计算方式 | 用途 |
|------|---------|------|
| 总字数/字符数 | `len([]rune(content))` | 样本规模参考 |
| 句长分布 | 按 `。！？.!?\n` 切句，分短(<15字)/中(15-30)/长(>30)三档，算占比 | 判断该风格的句子节奏倾向 |
| 平均句长 | 总字数 / 句子数 | 同上，单值概览 |
| 逗号密度 | 逗号出现次数 / 总字符数 | 判断流水句 vs 短句喜好 |
| 句号/感叹号/问号密度 | 各自出现次数 / 总字符数 | 判断语气倾向——陈述型/感叹型/反问型 |
| 引号密度 | 引号出现次数 / 总字符数 | 判断对话占比 |
| 段落数/平均段长 | 按 `\n\n` 切段统计 | 判断段落节奏——密集 vs 舒朗 |

这些数据作为结构化信息嵌入 ExtractStyle 的 prompt，让 LLM 在数据基础上做归纳，而非凭空猜测。

## API

所有 API 均为 Wails binding，前端通过 `useApp()` hook 调用。

### ListStyleSamples

```
ListStyleSamples(novelID int64) → []StyleSampleMeta
```

返回所有素材的摘要列表，不含全文。每个 meta 包含：
- `id`：素材唯一标识
- `name`：素材名称
- `preview`：正文前 120 字预览
- `tags`：标签列表
- `word_count`：字数
- `created_at`：创建时间

### CreateStyleSample

```
CreateStyleSample(CreateStyleSampleInput) → StyleSampleMeta
```

创建新素材。输入包含 novelID、name、content。写入文件系统 + git 提交。

### GetStyleSample

```
GetStyleSample(novelID int64, id string) → Sample
```

获取单条素材完整内容（含全文），用于详情编辑弹窗。

### UpdateStyleSample

```
UpdateStyleSample(UpdateStyleSampleInput) → void
```

更新素材的名称、标签、正文。直接覆写文件。

### DeleteStyleSample

```
DeleteStyleSample(DeleteStyleSampleInput) → void
```

删除指定素材文件 + git 提交。

### ComputeStyleStats

```
ComputeStyleStats(ComputeStyleStatsInput) → Stats
```

对指定素材列表进行确定性文本统计。输入为 novelID + sampleIDs 数组，仅统计选中的素材。

## Frontend

### 入口

ActivityBar（左侧图标导航栏）新增 Palette 图标「风格素材」。点击后主区域切换为 StyleSampleView，独占整个主区域（不含 Monaco 编辑器）。

### 卡片设计

玻璃拟态风格，灵感来自现代 UI 趋势：

- 24px 大圆角 `rounded-[24px]`
- 半透明背景 + 模糊 `bg-card/80 backdrop-blur-2xl`
- 选中态：`ring-2 ring-primary/40` + 蓝色光晕 `shadow-[0_8px_32px_rgba(14,165,233,0.12)]`
- 默认态：`border-white/15`，hover 时微上浮 `-translate-y-0.5`

每张卡片展示：
- **名称**：顶部标题，粗体
- **预览**：正文前 120 字，最多 3 行 `line-clamp-3`
- **标签**：渐变 pill 样式 `bg-gradient-to-r from-primary/10 to-accent/10`
- **元信息**：字数 + 创建时间，底部小字
- **操作**：右上角勾选框（选中态蓝色填充 + Check 图标）+ hover 时显示删除按钮

点击卡片 → 打开详情编辑弹窗。点击勾选框或删除按钮不会触发弹窗（`stopPropagation`）。

### 详情编辑弹窗

- 900px 宽 × 88vh 高，遮罩层半透明黑色
- 三个编辑字段：名称（input）、标签（input，逗号分隔）、正文（textarea）
- 正文区域用等宽字体 `font-mono`，方便查看排版细节
- 保存按钮调用 `UpdateStyleSample`

### 添加素材面板

展开式面板，内有名称 input + 正文 textarea。提交后自动收起，素材出现在卡片网格中。

### 提取流程

1. 用户在卡片网格中勾选素材（可多选）
2. 顶部工具栏显示已选数量 + 模型选择器（复用 PopSelect）+「开始提取」按钮
3. 拼接选中素材内容，调用 `ExtractStyle`
4. 预览生成的 Skill，可保存到 `skills/` 目录

## 与现有功能的关系

| 功能 | 输入 | 产物 | 适用场景 |
|------|------|------|---------|
| 短文提取（原） | 临时粘贴一段样文 | 仿写 Skill | 快速试一篇文章的风格 |
| 风格素材库（新） | 持久积累的多段素材 | 基于选中素材的仿写 Skill | 长期跟踪、持续优化某风格 |

两者共用同一个 `ExtractStyle` 后端逻辑，区别仅在于输入来源和持久化方式。短文提取的 tab 保留不变，作为快速入口。
